# SYSTEM SPECIFICATION: BOUTIQUE GRID & LAYOUT STANDARDS
**Project:** Personal Portfolio / Digital Temple (Premium E-commerce Services)  
**Design Philosophy:** Warm Minimalism / Editorial Layout / Structural Integrity

## 1. GLOBAL GEOMETRY (THE CONTAINER)
Все блоки сайта должны быть обернуты в единую систему контейнеров для соблюдения визуальной иерархии.

* **Max-Width:** Жесткий лимит контентной области — **1440px**. 
* **Safe Zones (Horizontal Padding):**
    * **Desktop ($window > 1024px$):** `px-20` (80px). Контент никогда не касается краев браузера.
    * **Mobile ($window < 768px$):** `px-5` (20px). Бутиковый стандарт плотности.
* **Vertical Rhythm (Section Spacing):**
    * **Macro-padding:** Каждая секция (1–14) разделяется отступом `py-40` (160px) на десктопе и `py-20` (80px) на мобильных. Это создает «дыхание» между смыслами.

## 2. THE 12-COLUMN GRID (STRUCTURAL LAW)
Весь контент, кроме блоков с принудительным центрированием (Hero, Manifesto), подчиняется закону асимметричной сетки **$4 + 8 = 12$**.

* **Logic:** Сетка `grid-cols-12` внутри `max-w-7xl`.
* **The Meta Zone (Col 1–4):** Левая часть экрана (33%) резервируется под «служебную» информацию: нумерация блока, микро-заголовки (маркеры), липкие заголовки.
* **The Content Zone (Col 5–12):** Правая часть экрана (66%) резервируется под основной контент: описание услуг, карточки портфолио, аккордеоны FAQ.
* **Negative Space:** Если блок не требует 8 колонок контента, мы оставляем пустые колонки внутри Content Zone. Пустота — это элемент дизайна, а не ошибка.

## 3. TYPOGRAPHY & READABILITY
* **Line Length Control:** Максимальная ширина текстового блока (абзаца) — **800px** (`max-w-3xl` или `max-w-prose`). Длинные строки убивают премиальность.
* **Alignment:** * Заголовки в Meta Zone прижаты к левому краю.
    * Контент в Content Zone выровнен по левому краю (Left Aligned) для удобства чтения. 
    * **Исключение:** Центрирование разрешено только для блоков 1 (Hero) и 10 (Manifesto).

## 4. MATHEMATICAL SPACING (8PT GRID SYSTEM)
Запрещено использование произвольных значений `px`. Все отступы внутри компонентов (`gap`, `margin`, `padding`) должны быть кратны **8** и использовать стандартные классы Tailwind CSS:

* **Micro:** `space-x-2` (8px), `space-x-4` (16px).
* **Small:** `gap-8` (32px).
* **Medium:** `gap-12` (48px) или `gap-16` (64px).
* **Large:** `gap-24` (96px).

## 5. VISUAL POLISH & TACTILITY
Каждый компонент должен поддерживать глобальные визуальные настройки:
* **Borders:** Только ультра-тонкие линии `border-zinc-800/10` (для светлых блоков) или `border-white/5` (для темных).
* **Glassmorphism:** При использовании `backdrop-blur`, степень размытия — `md` (12px), прозрачность фона — не более 80%.
* **Rounding:** Использовать большие радиусы для интерактивных элементов (`rounded-2xl` или `rounded-3xl`).

## 6. AGENT EXECUTION RULES
1.  **Strict Mode:** Перед генерацией кода агент должен проверить, соответствует ли разметка сетке 4/8.
2.  **No Inlining:** Все специфические настройки (анимации, кастомные сетки) выносятся в Tailwind классы или CSS переменные.
3.  **Refactoring Goal:** При обновлении "альфа-версий" блоков, агент обязан в первую очередь выровнять их согласно данной «Конституции», удаляя хаотичные отступы.
