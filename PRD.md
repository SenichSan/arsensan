# Product Requirements Document (PRD)

**Проект:** Личный сайт и интерактивное портфолио premium-разработчика e-commerce. **Версия документа:** 1.0

## 1. Executive Summary (Общее видение)

Проект представляет собой высококонверсионный гибридный веб-ресурс (Лендинг + SPA-портфолио). Его главная задача — продавать услуги по разработке кастомных, нишевых e-commerce решений через демонстрацию эталонного технического и визуального качества. Сайт должен служить физическим доказательством тезиса: «Я создаю премиальные, быстрые и безотказные цифровые продукты».

### 1.1. Целевая аудитория и User Personas

- **Персона А (Бизнес / Владельцы бутиков):** Ищут статус, увеличение продаж, отстройку от массмаркета. Боятся технических сбоев, сорванных сроков и «костылей». Не понимают в коде, но отлично считывают эстетику, плавность работы и метрики бизнеса.
    
- **Персона Б (Tech Leads / HR в IT):** Ищут senior-специалиста с глубоким пониманием современной архитектуры фронтенда и бэкенда. Оценивают чистоту анимаций, семантику, доступность (a11y), метрики производительности и стек технологий.

## 2. Архитектура и Технологический стек (Technical Specs)

Система строится по принципам Jamstack / Headless для обеспечения максимальной скорости и безопасности.

### 2.1. Frontend Core

- **Framework:** Next.js (App Router) — для обеспечения Server-Side Rendering (SSR) кейсов портфолио и статической генерации (SSG) основных блоков.
    
- **Язык:** TypeScript (Strict mode) — для строгой типизации и минимизации runtime-ошибок.
    
- **State Management:** React Context API (для управления состоянием "Интерактивного чекаута/корзины" и UI-темами).
    

### 2.2. Styling & UI

- **Стилизация:** Tailwind CSS + кастомные CSS Modules для сложных изоляций (например, в кинетической типографике).
    
- **UI-компоненты и a11y:** Использование **shadcn/ui** (базируется на невидимых логических примитивах Radix UI). Это гарантирует проекту 100% доступность (a11y), идеальную работу со скринридерами и навигацией с клавиатуры для сложных узлов (модальные окна, аккордеоны, селекты в чекауте). При этом сохраняется абсолютная свобода кастомизации дизайна без перебивания дефолтных стилей.
    

### 2.3. Motion & WebGL (Вау-эффект)

- **Complex Animations:** GSAP (ScrollTrigger, Flip) — для оркестрации сложных секвенций при скролле.
    
- **Smooth Scroll:** Lenis (от студии Studio Freight) — обязательное условие для премиального ощущения физики скролла.
    
- **3D Elements:** Three.js + React Three Fiber (R3F) — для рендеринга интерактивных объектов на главном экране (например, абстрактные формы, реагирующие на курсор). 3D-объекты должны быть из матового стекла, мягких органических форм (ткань, софт-боди) в цветах базовой палитры. Никакого холодного "киберпанка".
    

### 2.4. Backend & Content Management

- **CMS (Headless):** Sanity.io. База данных для управления кейсами в портфолио. Позволит легко добавлять новые проекты без изменения исходного кода.
    
- **Forms/Leads:** Нативные API-роуты Next.js + интеграция с Telegram API (для мгновенных уведомлений о заявках) и Resend/Nodemailer для автоотбивок на email клиента.
    

### 2.5. Метрики качества (KPIs)

- **Performance:** 95-100 баллов в Google Lighthouse (Desktop & Mobile).
    
- **Accessibility:** 100 баллов. Управление с клавиатуры (Tab-навигация), корректные `aria-labels` для всех 3D-элементов и анимаций.
    
- **SEO:** Динамические Meta-теги, OpenGraph изображения для каждого кейса, валидная микроразметка Schema.org.

## 3. Детальная архитектура интерфейса (Page Blocks Flow)

### Блок 1: Hero Section (Импульс)

- **Визуал:** Полноэкранный WebGL-канвас. Интерактивная типографика, которая искажается или подсвечивается при движении курсора.
    
- **Копирайт:** «Разработка премиальных интернет-магазинов, которые продают статус и генерируют прибыль».
    
- **Функционал:** Плавающая плашка «Свободен для 1 проекта в марте». Кнопка CTA с hover-эффектом (магнитная кнопка).
    

### Блок 2 & 3: Идентификация и Решение (Логика)

- **Визуал:** Смена фона (например, со светлого на светло-серый через GSAP ScrollTrigger) для контраста.
    
- **Структура:** Слева — перечисление болей (шаблоны, костыли). Справа — твое элегантное решение (кастом, надежность).
    
- **Анимация:** Плавное появление текста (Fade in up) по мере скролла.
    

### Блок 4: Услуги (Декомпозиция)

- **Визуал:** Аккордеон или интерактивные карточки (Bento Grid).
    
- **Контент:** UI/UX Design, Frontend (Motion/WebGL), Headless Backend, Performance & SEO.
    
- **Интерактив:** При наведении на услугу меняется глобальный фон или 3D-объект рядом демонстрирует суть услуги.
    

### Блок 5: Портфолио (Доказательства)

- **Визуал:** Горизонтальный скролл (GSAP) или каскадная сетка.
    
- **Структура карточки:** Крупный мокап + Теги (Next.js, E-commerce) + Метрики (Конверсия +40%, Загрузка 0.8s).
    
- **Функционал:** View Transitions API. При клике на кейс он плавно расширяется на весь экран без перезагрузки, показывая подробный Case Study (Задача -> Решение -> Результат).
    

### Блок 6: Стек технологий (Tech Radar)

- **Визуал:** Бесконечная плавная бегущая строка (Marquee) или интерактивный грид с логотипами технологий. Монохромный дизайн, чтобы не отвлекать от сути.
    

### Блок 7 & 8: Трансформация и Процесс

- **Визуал:** Таймлайн в виде вертикальной линии, которая закрашивается по мере скролла пользователя.
    
- **Шаги:** 1. Discovery (Стратсессия), 2. UX/UI, 3. Dev & Motion, 4. QA & Testing, 5. Launch.
    

### Блок 9 & 10: Социальные доказательства и Манифест

- **Визуал:** Крупная акцидентная типографика (цитаты клиентов). Фото разработчика в стилистике editorial (черно-белое или с глубокими тенями).
    
- **Контент:** Честный рассказ о подходе "Бутик для бутиков".
    

### Блок 11: Интерактивный Checkout (Главный CTA)

- **Визуал:** Имитация корзины премиум-магазина.
    
- **Флоу:**
    
    1. _Шаг 1 (Выбор):_ Пользователь выбирает нужные модули (чекбоксы: "Дизайн", "Разработка", "Интеграции").
        
    2. _Шаг 2 (Корзина):_ Подсчет примерного бюджета (опционально) или просто статус "В корзине".
        
    3. _Шаг 3 (Checkout):_ Форма ввода (Имя, Бренд/Ссылка, Контакты).
        
    4. _Шаг 4 (Success):_ Кастомная страница благодарности с анимацией "Заказ оформлен, готовьтесь к трансформации".
        
- **Технологии:** React Hook Form + Zod для жесткой валидации.

## 4. UX/UI Guidelines (Правила дизайна)

-Вот как выстроена визуальная стратегия этого концепта:

### 1. Общее настроение и атмосфера (Премиальная тактильность)

Сайт транслирует уверенность, экспертность и дороговизну, но без холодного снобизма.

- **От off.site** мы взяли журнальную верстку, обилие «воздуха» (негативного пространства) и смелую подачу смыслов крупными формами.
    
- **От yummygum.com** — ощущение «тактильности» и дружелюбия интерфейса. Интерфейс не плоский, он кажется физически осязаемым благодаря мягкости форм.
    
- **От drewl.com** — структурную динамику. Сайт ведет пользователя по истории, ритмично меняя светлые и темные блоки, чтобы удерживать внимание.
    

### 2. Цветовая палитра (Теплый минимализм)

Я отказался от стандартного белоснежного фона (`#FFFFFF`) и типичного «айтишного» синего в пользу более сложной и благородной гаммы:

- **База (Фон):** Оттенок слоновой кости / теплый светло-серый (в концепте это палитра `Stone`). Это убирает эффект «выжженной сетчатки» и создает ощущение крафтовости, дорогой бумаги.
    
- **Типографика (Текст):** Графитовый, почти черный цвет. Он дает отличный контраст для чтения, но не режет глаз.
    
- **Акцент (Кнопки, выделения, графики):** Сложный цвет Индиго (глубокий сине-фиолетовый). Он ассоциируется с технологичностью, диджиталом и экспертизой, но выглядит свежее и дороже стандартного синего.
    
- **Визуальный перегруз (Инверсия):** Блок чекаута специально сделан темным. Этот резкий контраст (как у drewl.com) работает как визуальная пауза: он сбрасывает замылившийся взгляд пользователя и фокусирует его на главном интерактивном элементе — выборе услуг и конверсии.
    

### 3. Типографика (Характер + Функция)

Визуальный стиль во многом строится на шрифтовом контрасте:

- **Акцентный шрифт (Заголовки):** Используется геометричный, немного техничный гротеск (подразумевается _Space Grotesk_). Он привлекает внимание, выглядит ультрасовременно и задает характер бренда — инновационный и смелый.
    
- **Наборный шрифт (Тексты, интерфейс):** Максимально нейтральный и читабельный неогротеск (например, _Inter_). Его задача — не отвлекать от смысла и легко читаться в любом размере.
    

### 4. Формообразование и UI-элементы (Мягкость и Глубина)

Здесь концепт сильно опирается на подход _Yummygum_:

- **Крупные скругления:** Карточки, кнопки и контейнеры имеют большие радиусы скругления. Это делает интерфейс визуально «безопасным» и современным. Никаких агрессивных острых углов.
    
- **Glassmorphism (Эффект матового стекла):** Верхняя панель навигации и подложки под карточками имеют легкую полупрозрачность и эффект размытия фона. Это создает многослойность и легкость — элементы как бы парят над основным фоном.
    
- **Мягкие тени:** Вместо резких черных теней используются очень объемные, размытые и полупрозрачные тени, которые создают эффект свечения или естественного отбрасывания света.
    

### 5. Интерактивность как часть дизайна

Визуальный опыт не статичен. Взаимодействие с сайтом должно быть таким же плавным, как сам дизайн:

- Взаимодействие с чекбоксами в чекауте и пересчет статуса корзины происходят синхронно и плавно, давая пользователю ощущение полного контроля над сборкой проекта.
    
- Переключение вкладок (в блоке процессов) происходит через мягкое затухание (fade-in), а не резкие скачки контента.
    
- Ползунки калькулятора и перестроение линий на графике работают синхронно, давая пользователю ощущение полного контроля над данными.

## **5. Roadmap реализации (План разработки)**

- **Фаза 1: Проектирование (Неделя 1)**
    
    - Сбор мудборда и референсов.
        
    - Проектирование wireframes (Figma).
        
    - Написание финального копирайтинга для всех блоков.
        
- **Фаза 2: Базовая архитектура (Неделя 2)**
    
    - Инициализация Next.js проекта, настройка Tailwind, ESLint, Prettier.
        
    - Настройка базового роутинга и Headless CMS (создание схем для портфолио).
        
- **Фаза 3: Разработка логики и верстка (Неделя 3-4)**
    
    - Верстка всех компонентов (Pixel Perfect).
        
    - Программирование логики "Интерактивного чекаута".
        
    - Подключение API для отправки заявок.
        
- **Фаза 4: Анимации и WebGL (Неделя 5)**
    
    - Интеграция Lenis (Smooth Scroll).
        
    - Настройка GSAP таймлайнов для каждого логического блока.
        
    - Разработка 3D-сцены для Hero секции.
        
- **Фаза 5: Оптимизация и Релиз (Неделя 6)**
    
    - Аудит Lighthouse, оптимизация изображений (Next/Image), lazy loading.
        
    - Настройка SEO (Meta, Sitemap, Robots.txt).
        
    - Деплой на Vercel, привязка домена, финальное тестирование.
